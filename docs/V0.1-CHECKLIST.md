# v0.1 Checklist

v0.1 target: fresh node boot -> pair -> connect link -> screen appears, with no terminal use for the user.

## Implemented In Repo

- Cargo workspace and locked monorepo layout scaffold.
- `sw-core` protocol/config/pairing/capability types.
- Real UUID validation and generation for node identity.
- `sw-agent` VA-API render-device enumeration and H.264 decode detection.
- `sw-agent` persistent identity store driven by external config.
- In-process agent API router:
  - `GET /v1/health`
  - `GET /v1/capabilities`
  - `GET /v1/pairing`
  - `POST /v1/pairing`
- Pairing state model with explicit unavailable/waiting/paired states.
- Generated and persisted node certificate material from configured paths.
- Node certificate fingerprint exposed through pairing state.
- Companion-facing route constants in `sw-core`.
- Product-boundary docs for host and node UX.

## Non-Negotiable UX Rule

Neither side should expose upstream setup as the product:

- Host: users use the SecondWind companion, not Apollo's browser dashboard.
- Node: users use the SecondWind node image, not Debian/Moonlight/cage/driver commands.

Debug logs may mention upstream internals for support, but normal product flows must explain issues in SecondWind terms.
